## 1. 项目概述

### 1.1 背景
贪吃蛇是 2D 游戏领域的"Hello World"，核心逻辑简单但涉及游戏循环、状态机、碰撞检测、输入处理等通用概念。是网页游戏开发入门的最佳练手项目。

### 1.2 目标
- **学习目标**：掌握 Canvas 2D 渲染、游戏主循环、键盘事件、碰撞检测、数据结构（队列）。
- **产品目标**：交付一款可玩、可上线的网页版贪吃蛇小游戏。

### 1.3 范围
**包含**：单模式（经典模式）、键盘控制、本地最高分记录。
**不包含**：多人对战、网络排行榜、复杂皮肤系统。

## 3. 核心玩法

### 3.1 玩法描述
玩家控制一条蛇在矩形地图上移动，通过吃食物让身体变长。蛇头撞到墙壁或自己的身体时游戏结束。目标是吃到尽可能多的食物、得到尽可能高的分数。

### 3.2 胜负条件
- **胜利**：无固定胜利条件，以分数最高为目标（生存时间越长分越高）。
- **失败**：蛇头撞墙 或 撞到自身任意部位。

### 3.3 计分规则
| 行为 | 得分 |
| --- | --- |
| 吃到 1 个普通食物 | +10 分 |
| 每存活 1 秒（可选） | +1 分 |

## 6. UI/UX 设计

### 6.1 界面布局
```
┌────────────────────────────────┐
│   Snake              ⭐ 1230   │   ← 顶部：标题 + 分数
├────────────────────────────────┤
│                                │
│                                │
│        ┌──┐                    │
│        │● │ ← 蛇头             │
│        └──┘                    │   ← 中部：游戏画布
│           ↓                    │      (默认 20×20 网格)
│        ┌──┐                    │
│        │■ │                    │
│        └──┘                    │
│                                │
│                ┌──┐           │
│                │★ │ ← 食物    │
│                └──┘           │
│                                │
├────────────────────────────────┤
│  ↑ ↓ ← → / WASD 移动  │ 空格暂停 │  ← 底部：操作提示
└────────────────────────────────┘
```

### 6.2 视觉规范
- **画布**：400×400 px（每个网格 20×20 px，共 20×20 网格）。
- **配色**：背景 `#1a1a2e`，蛇身 `#4ade80`，蛇头 `#22c55e`，食物 `#f87171`。
- **字号**：分数 24px，提示文字 14px。

### 6.3 动效
- 吃食物时身体节有 0.1s 缩放弹跳。
- 游戏结束全屏渐变红色 0.5s。
- 分数变化时数字短暂放大 1.2 倍。

## 8. 数据结构关键设计

### 8.1 蛇（Snake）
```ts
// 用数组模拟蛇身，数组头部是蛇头
type Point = { x: number; y: number };
class Snake {
  body: Point[];          // 蛇身坐标队列
  direction: Direction;   // 当前方向
  nextDirection: Direction; // 缓冲下一次方向（防止同帧反向）
}
```

### 8.2 食物（Food）
```ts
class Food {
  position: Point;        // 食物坐标
  type: 'normal' | 'golden';
  spawnTime?: number;     // 金色食物的过期时间
}
```

### 8.3 游戏状态
```ts
enum GameState {
  MENU = 'menu',
  PLAYING = 'playing',
  PAUSED = 'paused',
  GAME_OVER = 'game_over'
}
```

## 9. 技术约束

- **渲染**：Canvas 2D（不使用 WebGL，避免过度设计）。
- **存储**：localStorage 存储最高分，容量上限 5MB 远足够。
- **构建**：Vite 5.x，TypeScript 5.x 严格模式。
- **依赖**：仅核心库（Phaser 等暂不引入），保持纯原生。
- **浏览器目标**：Evergreen Browsers + 移动端 Safari/Chrome。

---

## 10. 风险与依赖

| 风险 | 影响 | 应对 |
| --- | --- | --- |
| 输入抖动（快速按键导致 180° 反向） | 中 | 引入 `nextDirection` 缓冲，渲染前统一应用 |
| 移动端无键盘 | 中 | V2 增加触屏滑动控制 |
| 长时间游玩后帧率下降 | 低 | Canvas 2D 已足够，无需额外优化 |
