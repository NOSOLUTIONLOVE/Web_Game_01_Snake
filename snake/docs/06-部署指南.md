## 二、方案 A：Vercel 部署（推荐）

### A.1 通过 Vercel 网站（最简单）

**前置**：GitHub 账号 + Vercel 账号（用 GitHub 登录即可）

#### 步骤

1. **推送代码到 GitHub**
   ```bash
   cd D:\Project\Web_Game_01\games\snake
   git init
   git add .
   git commit -m "feat: MVP 完成 v1.0.0"
   git branch -M main
   git remote add origin https://github.com/<你的用户名>/snake-web.git
   git push -u origin main
   ```

2. **在 Vercel 导入项目**
   - 访问 https://vercel.com/new
   - 选择 **Import Git Repository**
   - 找到 `snake-web` 仓库，点击 **Import**

3. **配置项目**
   - **Framework Preset**：自动识别为 `Vite`
   - **Build Command**：`npm run build`（默认）
   - **Output Directory**：`dist`（默认）
   - **Install Command**：`npm install`（默认）

4. **部署**
   - 点击 **Deploy** 按钮
   - 等待 1-2 分钟
   - 部署成功后会显示 🎉 和一个 URL，形如 `https://snake-web-xxx.vercel.app`

5. **（可选）绑定自定义域名**
   - Project Settings → Domains
   - 添加你的域名，按提示配置 DNS

---

### A.2 通过 Vercel CLI（进阶）

适合想从命令行直接部署的开发者。

#### 步骤

1. **安装 Vercel CLI**
   ```bash
   npm install -g vercel
   ```

2. **登录 Vercel**
   ```bash
   vercel login
   ```
   按提示选择登录方式（GitHub 推荐）。

3. **首次部署（预览环境）**
   ```bash
   cd D:\Project\Web_Game_01\games\snake
   vercel
   ```
   - 会询问项目配置（直接回车用默认值即可）
   - 部署完成会显示 Preview URL

4. **部署到生产环境**
   ```bash
   vercel --prod
   ```
   - 拿到生产 URL：`https://snake-web-xxx.vercel.app`

5. **（可选）查看部署状态**
   ```bash
   vercel ls
   ```

## 三、方案 B：Cloudflare Pages 部署（国内访问友好）

### B.1 通过 Cloudflare Dashboard

1. **推送代码到 GitHub**（同 A.1 步骤 1）

2. **登录 Cloudflare Pages**
   - 访问 https://pages.cloudflare.com/
   - 点击 **Create a project** → **Pages** → **Connect to Git**

3. **选择仓库**
   - 选择 `snake-web` 仓库
   - 点击 **Begin setup**

4. **配置构建**
   - **Project name**：`snake-web`
   - **Production branch**：`main`
   - **Build command**：`npm run build`
   - **Build output directory**：`dist`
   - **Environment variables**：（无）

5. **保存并部署**
   - 点击 **Save and Deploy**
   - 等待几分钟
   - 拿到 URL：`https://snake-web.pages.dev`

### B.2 Cloudflare 优势

- ✅ 国内访问速度快（节点多）
- ✅ 无限带宽
- ✅ 免费 HTTPS
- ❌ 国内仍未备案的域名需要 Cloudflare 接入

## 四、方案 C：GitHub Pages 部署（零成本）

### C.1 通过 GitHub Actions 自动部署

**前置**：GitHub 账号

#### 步骤

1. **推送代码到 GitHub**（同 A.1 步骤 1）

2. **创建 GitHub Actions 工作流**
   
   在项目根目录创建 `.github/workflows/deploy.yml`：
   ```yaml
   name: Deploy to GitHub Pages
   
   on:
     push:
       branches: [main]
   
   jobs:
     build-and-deploy:
       runs-on: ubuntu-latest
       steps:
         - uses: actions/checkout@v4
         
         - uses: actions/setup-node@v4
           with:
             node-version: 20
             cache: 'npm'
         
         - run: npm ci
         - run: npm run build
         
         - uses: peaceiris/actions-gh-pages@v3
           with:
             github_token: ${{ secrets.GITHUB_TOKEN }}
             publish_dir: ./dist
   ```

3. **配置 Vite base 路径**
   
   修改 `vite.config.ts`：
   ```ts
   export default defineConfig({
     base: '/snake-web/', // ← 改成你的仓库名
     // ...
   });
   ```

4. **启用 GitHub Pages**
   - 仓库 Settings → Pages
   - Source 选 **Deploy from a branch**
   - Branch 选 `gh-pages` / `/root`
   - 保存

5. **等待 Actions 完成**
   - 几分钟后访问：`https://<用户名>.github.io/snake-web/`

## 七、常见问题

### Q1: 部署后页面空白？

A：通常是 base 路径不对。
- 检查 `vite.config.ts` 的 `base` 配置
- Vercel/Cloudflare 默认相对路径 `./` 即可
- GitHub Pages 需要改成 `/<仓库名>/`

### Q2: 静态资源 404？

A：同上，base 路径问题。
- 打开 DevTools → Network 看哪些资源 404
- 调整 `vite.config.ts` 的 `base`

### Q3: 第一次部署很慢？

A：Vercel/Cloudflare 首次需要安装依赖（npm install），约 1-2 分钟。后续会缓存 node_modules，几秒即可。

### Q4: 国内访问 Vercel 慢？

A：换 Cloudflare Pages，国内访问友好。

### Q5: 想用自定义域名？

A：
- **Vercel**：Project Settings → Domains → 添加 + 配置 DNS
- **Cloudflare**：Pages → Custom domains → 添加
- **GitHub Pages**：Settings → Pages → Custom domain
