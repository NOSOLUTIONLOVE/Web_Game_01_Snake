## 一、项目背景

### 1.1 立项动机
贪吃蛇是游戏史上最经典的品类之一，自 1976 年首次出现以来已迭代出无数变种。选择它作为本项目库（Web_Game_01）的第一款产品，原因有三：

1. **技术代表性**：贪吃蛇虽简单，但涵盖了网页游戏开发的所有核心概念 —— 游戏主循环、状态机、碰撞检测、输入处理、持久化。
2. **完成度高**：1-2 天即可交付完整可玩游戏，建立开发信心。
3. **可复用模板**：项目骨架（脚手架 + 渲染层 + 状态机）可复用于后续项目（2048、俄罗斯方块等）。

### 1.2 项目定位
- **类型**：2D 网页休闲游戏
- **风格**：经典复刻 + 现代 UI 修饰
- **目标平台**：桌面浏览器（Chrome / Edge / Firefox）+ 移动端浏览器

## 二、项目目标

### 2.1 学习目标（核心）
通过本项目掌握以下网页游戏开发基础能力：

| 序号 | 能力 | 衡量标准 |
| --- | --- | --- |
| 1 | Canvas 2D 渲染 | 能独立绘制游戏画面 |
| 2 | 游戏主循环 | 理解 requestAnimationFrame 与固定时间步长 |
| 3 | 状态机 | 至少 4 个游戏状态（MENU/PLAYING/PAUSED/GAME_OVER） |
| 4 | 输入系统 | 键盘 + 触屏双端支持 |
| 5 | 碰撞检测 | AABB 碰撞检测 |
| 6 | 数据持久化 | localStorage 存最高分 |

### 2.2 产品目标
- ✅ 交付一款可在浏览器中流畅运行的贪吃蛇游戏
- ✅ 实现 PRD-01 中全部 MVP 功能（F-01 至 F-10）
- ✅ 完成至少 3 项 V2 扩展（难度递增 / 音效 / 触屏）
- ✅ 可通过 Vercel 一键部署

### 2.3 质量目标

| 指标 | 目标 |
| --- | --- |
| 性能 | 稳定 60fps，蛇移动间隔 ≤ 200ms |
| 兼容性 | Chrome 100+ / Edge 100+ / Firefox 100+ / Safari 15+ / 移动端主流浏览器 |
| 代码质量 | TypeScript 严格模式，零 `any`，ESLint 通过 |
| 体积 | 首屏加载 ≤ 200KB（gzip） |
| 可用性 | 3 秒内可上手，无需教程 |

## 三、范围定义

### 3.1 ✅ 包含范围

| 模块 | 范围 |
| --- | --- |
| 游戏玩法 | 单模式经典贪吃蛇 |
| 控制方式 | 键盘（方向键/WASD）+ 触屏滑动 |
| 数据存储 | localStorage（最高分） |
| UI | 菜单 / 游戏中 / 暂停 / 结束 四个场景 |
| 视觉 | 经典绿蛇 + 红食物配色 |
| 反馈 | 基础动画 + 音效（可选） |

### 3.2 ❌ 不包含范围

| 不包含 | 原因 |
| --- | --- |
| 多人对战 | 单机练手项目，复杂度超出范围 |
| 网络排行榜 | 无后端依赖 |
| 皮肤商城 | 商业化功能，超出 MVP 范围 |
| 关卡编辑器 | 贪吃蛇本身不需要关卡 |
| 成就系统 | 不在 V1 范围 |

## 五、风险评估

| 风险 | 概率 | 影响 | 应对措施 |
| --- | --- | --- | --- |
| 输入抖动导致 180° 反向自撞 | 高 | 中 | 引入 `nextDirection` 缓冲机制 |
| 移动端无键盘无法操作 | 中 | 中 | V2 必加触屏滑动控制 |
| Canvas 帧率波动 | 低 | 中 | 使用固定时间步长逻辑更新 |
| 长时间游玩 localStorage 溢出 | 极低 | 低 | 仅存储最高分 + 时间戳 |

## 六、资源投入

### 6.1 人力
- **1 人全栈开发**（前端为主，可独立完成）

### 6.2 时间
- **总工期**：1.5 - 2 天（按每天 6-8 小时有效开发计）

### 6.3 技术资源
- Node.js 18+ / npm 9+
- Vite 5.x
- TypeScript 5.x
- 浏览器开发者工具
- Vercel 账号（免费）

### 6.4 第三方资源
- 字体：Google Fonts（`Press Start 2P`，可选）
- 图标：Lucide Icons / Emoji 替代
- 音效：开源 CC0 音效（如 freesound.org）
