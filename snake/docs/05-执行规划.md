## 二、总体策略

### 2.1 核心策略：分阶段交付

**绝不一上来写完所有代码**。贪吃蛇虽小，但一次性写完容易出"中间某步错了自己都不知道"的问题。分阶段好处：

| 好处 | 说明 |
| --- | --- |
| **可演示** | 每完成一阶段都有一个能玩的版本 |
| **可验证** | 范围明确，问题易定位 |
| **可回滚** | 单阶段出问题不影响其他阶段 |
| **心理反馈** | 不会觉得"还没做完" |

### 2.2 6 个阶段划分

```
P0 脚手架 ──► P1 蛇能跑 ──► P2 通关闭环 ──► P3 MVP ──► P4 体验优化 ──► P5 部署
  1-2h          1-2h            2-3h           1-2h          1-2h           1h
```

### 2.3 阶段依赖关系

```
P0（必须最先）
  │
  ├──► P1 ──► P2 ──► P3 ═══► P4 ═══► P5
              │      │       │       │
              │      │       │       └──► 部署上线
              │      │       └──► 体验打磨
              │      └──► MVP 完整
              └──► 核心玩法
```

## 四、关键风险与应对

### 风险 1：固定时间步长写错，蛇速不稳定

| 风险 | 概率 | 影响 |
| --- | --- | --- |
| 不同设备帧率不同导致蛇速差异 | 高 | 中 |

**应对**：
- 使用标准 accumulator 模式（参考《04-项目架构》Game.ts）
- 蛇移动走"逻辑 tick"而非"每帧"
- P1 阶段就要在低性能设备验证

### 风险 2：方向输入 180° 反向自撞

| 风险 | 概率 | 影响 |
| --- | --- | --- |
| 快速按反方向键导致游戏秒死 | 高 | 高（直接劝退） |

**应对**：
- 必须有 `nextDirection` 缓冲（P1 阶段必加）
- 每帧统一应用方向，而非每输入立即应用

### 风险 3：TypeScript 类型跨文件传导出问题

| 风险 | 概率 | 影响 |
| --- | --- | --- |
| 类型导入路径混乱 | 中 | 中 |

**应对**：
- 统一 `import type { ... } from '../core/Input'`
- Snake 等核心类先写类型 stub，再补实现

### 风险 4：首次部署 Vercel 配置坑

| 风险 | 概率 | 影响 |
| --- | --- | --- |
| base path 错误导致资源 404 | 中 | 中 |

**应对**：
- P0 阶段就把 `base: './'` 设好
- 部署前本地 `npm run preview` 验证

### 风险 5：移动端触摸与点击冲突

| 风险 | 概率 | 影响 |
| --- | --- | --- |
| 触屏滑动触发点击事件 | 中 | 中 |

**应对**：
- 用 `touchstart` + `touchend` 计算方向，不依赖 click
- click 事件只处理"确认"动作

## 五、提交与版本策略

### 5.1 Commit 粒度

每个 P 阶段拆成多个 commit：

```
P0: 1-2 个 commit（脚手架 + 入口）
P1: 2-3 个 commit（Snake 类 + Game 循环 + 输入）
P2: 3-4 个 commit（Food + 碰撞 + 计分 + UI）
P3: 2-3 个 commit（状态机 + 持久化）
P4: 1 个 commit / 子功能
P5: 1 个 commit（部署配置）
```

### 5.2 Commit 消息规范

```
feat: 实现蛇的移动逻辑
fix: 修复 180° 反向自撞
docs: 更新架构文档
chore: 调整 tsconfig 严格模式
```

### 5.3 分支策略（可选）

- 主分支 `main`：稳定版本
- 开发分支 `develop`：日常开发
- 功能分支 `feat/p1-snake-movement`：单功能

> 简单起见，MVP 阶段直接 main 分支就行。

## 六、测试策略

### 6.1 单元测试（P2 后引入）

测试核心算法（Vitest 即可，简单）：

```ts
// snake.test.ts
test('蛇不能 180° 反向', () => {
  const snake = new Snake();
  snake.direction = 'RIGHT';
  snake.queueDirection('LEFT');
  expect(snake.nextDirection).toBe('RIGHT');
});

// food.test.ts
test('食物不生成在蛇身上', () => {
  const snake = new Snake();
  const food = new Food();
  food.spawn(snake);
  const onSnake = snake.body.some(
    p => p.x === food.position.x && p.y === food.position.y
  );
  expect(onSnake).toBe(false);
});
```
